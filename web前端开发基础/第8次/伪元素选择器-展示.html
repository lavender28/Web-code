伪元素 Pseudo Element

在不修改 HTML 结构的前提下，凭空创造一个有内容的虚拟容器——这就是伪元素的魔法。

::before / ::after CONTENT IS KEY
01

从一个具体问题说起

想给段落首字母加颜色，传统做法要修改 HTML，伪元素让你省掉这一步。

— 传统做法 · BEFORE —

污染 HTML 结构

为了让首字母变红，必须在 HTML 中包一层 <span>，再给它配一个类名。 

<p>
  <span class="first">H</span>ello, World
</p>

.first { color: red; }
Hello, World
— 伪元素做法 · AFTER —

HTML 保持纯净

HTML 一个字都不用改，仅靠一行 CSS 就能实现完全相同的效果。这就是伪元素的核心价值。

<p>Hello, World</p>




p::first-letter { color: red; }
Hello, World
伪元素的本质，是让 CSS 拥有了 "凭空创造内容" 的能力。
本讲将解决三个核心问题：第一，伪元素和伪类到底有什么区别？第二，::before 和 ::after 怎么用？第三，为什么离开 content 属性，伪元素就完全失效？
02

伪类 vs 伪元素

这是最核心也最容易混淆的概念。一个比喻彻底讲清。

教室比喻：一群学生坐在教室里。要找"小明"，喊名字就行（基础选择器）；要找"第二排第三列的同学"，需要描述位置（这就是伪类）；如果学生本来不存在，老师凭空指着空位说"这是新来的小张"（这就是伪元素）。
— 伪类 PSEUDO-CLASS —

找已有的元素

伪类选中的是 DOM 中真实存在的元素，只是它具有某种特殊状态或处于特殊位置。

  • 用 单冒号 : 表示
  • 例如 :hover、:first-child
  • 不创造任何新元素

— 伪元素 PSEUDO-ELEMENT —

造一个新元素

伪元素并不在 HTML 中真实存在，它是 CSS 在渲染时虚拟创建的，可以为它添加内容和样式。

  • 用 双冒号 :: 表示（CSS3 规范）
  • 例如 ::before、::after
  • 必须配合 content 才能生成

维度 伪类（Pseudo-class） 伪元素（Pseudo-element）
选择对象 真实存在的元素 虚拟创建的内容
书写形式 单冒号 : 双冒号 ::（CSS3 规范）
核心用途 根据状态/位置筛选元素 凭空生成内容或装饰区域
典型例子 li:first-child · a:hover p::before · p::first-letter
是否需要 content 不需要 必须（::before / ::after）
03

::before 在元素前插入内容

点击下方开关，亲眼见证 ::before 如何在元素之前凭空创造内容。

STYLE.CSS 
.warn::before {
  content: "⚠ ";
  color: #c8392f;
  font-weight: 700;
}
INDEX.HTML 
<div class="warn">
  请注意磁盘空间不足
</div>




/* HTML 完全不变，
   仅靠 CSS 添加内容 */
实时渲染
无伪元素
::before 启用
关键观察：左侧是没有伪元素的原始效果，右侧是启用 ::before 后的效果。HTML 完全相同——所有的差异都来自 CSS 中的那几行 ::before 规则。
04

::after 在元素后插入内容

用法与 ::before 完全镜像，只是插入位置在元素之后。

STYLE.CSS 
a.external::after {
  content: " ↗";
  color: #8a7a6c;
  font-size: 0.85em;
}
INDEX.HTML 
<a class="external"
   href="https://...">
  访问外部资源
</a>
实时渲染
无伪元素
::after 启用
::before 与 ::after 是孪生兄弟——位置不同，逻辑一致。
05

content 属性的多种形态

没有 content，伪元素就不会生成。但 content 的值可以是文字、引号、URL、计数器等多种形式。

— 互动练习场 · 切换 content 的取值 —

目标元素：.target::before

CSS 选择器
.target::before {
  content: "【精选】";
  color: #c8392f;
}

content 属性的常见取值

取值形式 含义 典型用法
"文字" 插入字符串 content: "【新】";
"" 空字符串（非常常用） 纯装饰，配合宽高背景做图形
url(...) 插入图片 content: url("icon.svg");
attr(属性名) 读取元素的属性值 content: attr(data-tip);
counter(name) 插入 CSS 计数器 自动生成章节序号
open-quote / close-quote 引号 配合 quotes 属性使用
致命陷阱：没有 content 属性的 ::before 和 ::after 根本不会生成——哪怕你写了 width、height、background。即使不需要文字内容，也要写 content: "";（空字符串）作为占位。
06

单冒号还是双冒号？

CSS2 时代单双冒号通用，CSS3 规范要求伪元素使用双冒号。但兼容性考量下，二者今天都能用。

— CSS2 写法 —

:before / :after

单冒号写法，从 CSS2 时代沿用至今。兼容性更好，能支持老版本 IE 浏览器。

p:before {
  content: "→ ";
}
— CSS3 规范 —

::before / ::after

双冒号写法，CSS3 引入用于明确区分伪类与伪元素。语义更清晰，是现代规范。

p::before {
  content: "→ ";
}
对比项 单冒号 :before 双冒号 ::before
规范 CSS2 CSS3
兼容性 更好（含老版 IE） 现代浏览器良好
语义 与伪类形式重复，模糊 明确区分伪元素
建议 仅在需要兼容老 IE 时使用 HTML5+CSS3 项目首选 ✓
实际开发：在 HTML5 + CSS3 现代项目中，应当遵循规范使用双冒号写法。但要清楚两种写法在现代浏览器中渲染效果完全一致——遇到老代码用单冒号也别惊讶。
