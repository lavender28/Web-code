CSS3 结构化伪类选择器

通过左侧 CSS、中间 HTML 与右侧实时渲染的三栏对照，直观感受不同伪类的命中差异。被命中的元素会高亮显示。

:root 选择器

匹配文档的根元素（在 HTML 中即 <html> 标签）。最常见的用法是定义全局 CSS 变量。

style.css
:root {
  --main: #3498db;
  --size: 18px;
}

h3 {
  color: var(--main);
  font-size: var(--size);
}
index.html
<html>     <!-- :root 命中这里 -->
  <body>
    <h3>使用了 CSS 变量</h3>
  </body>
</html>

核心要点

:root 等同于选中 <html> 标签，但优先级更高。在它内部定义的 CSS 变量（如 --main）整个页面任何地方都可以通过 var(--main) 引用，是定义全局主题色、字号等设计 token 的最佳位置。

:not() 选择器

排除符合括号内选择器的元素。下方演示了 h3:not(.one) —— 选中所有不带 .one 类的 h3。

style.css
h3:not(.one) {
  background: 珊瑚红;
}
index.html
<h3>标题一</h3>      ✓ 命中
<h3 class="one">标题二</h3>  ✗ 排除
<h3>标题三</h3>      ✓ 命中

常见陷阱

切勿单独使用 :not(p) —— 它会匹配 <html>、<body> 在内的所有非 p 元素，再叠加 CSS 继承机制，最终连 <p> 自己都会受影响。:not() 应总与具体选择器组合使用，例如 h3:not(.one) 或 .container :not(p)，把作用范围限定在明确的子集里。

:only-child 选择器

选中作为父元素中唯一子元素的标签。如果父元素只有这一个孩子，就命中。

style.css
p:only-child {
  background: 高亮;
}
index.html
<div>
  <p>我是独生子</p>  ✓ 命中
</div>

<div>
  <p>段落 A</p>     ✗ 有兄弟
  <p>段落 B</p>     ✗ 有兄弟
</div>

判断逻辑

:only-child 是从父元素的角度判断的——只要父元素还有任何其他子元素（哪怕类型不同），就不再"唯一"。这与 :only-of-type 不同，后者只看同类型中是否唯一。

:first-child 与 :last-child

分别选中父元素的第一个和最后一个子元素。是制作导航分隔线、列表首尾特殊样式的经典选择器。

style.css
li:first-child {
  background: 高亮;
}

li:last-child {
  background: 高亮;
}
index.html
<ul>
  <li>苹果</li>  ✓ first
  <li>香蕉</li>
  <li>橙子</li>
  <li>葡萄</li>  ✓ last
</ul>

真实场景

导航栏中常见的"除最后一项外都加分隔线"效果，组合 :not() 即可优雅实现：nav a:not(:last-child) { border-right: 1px solid #ccc; }

:nth-child(n) 选择器

最强大的位置选择器，支持数字、关键字 odd/even、以及公式 an+b。试试切换下方按钮，看看不同写法的命中效果。

li:nth-child(3) {

公式 an+b 的运作方式

CSS 计算时让 n 从 0 开始依次代入（n=0,1,2,3...），所有得到的正整数即为命中位置。例如 3n+1：n=0→1, n=1→4, n=2→7，所以命中第 1、4、7 个。注意位置编号本身从 1 开始计数。

:first-of-type / :nth-of-type(n)

"of-type" 系列只关心同类型的兄弟元素，跳过其他类型的标签独立计数。

style.css
p:first-of-type {
  background: 高亮;
}
index.html
<div>
  <h3>标题</h3>
  <p>第一段</p>     ✓ 命中
  <p>第二段</p>
  <p>第三段</p>
</div>

关键差异

这里如果改用 p:first-child，将什么也选不中——因为父元素的第一个子元素是 <h3> 而不是 <p>。:first-of-type 跳过 <h3>，从第一个 <p> 开始计数。这一点请进入下一章节"⚔️ child vs of-type"做深度对比。

记忆口诀

child 系列看整体顺序——元素必须既是目标类型，又恰好处于该位置；
of-type 系列看同类顺序——只在同类型兄弟中独立计数，其他类型完全忽略。

:empty 选择器

选中没有任何子元素且不包含任何文本（包括空格）的标签。

style.css
div:empty {
  background: 高亮;
  height: 30px;
}
index.html
<div>有内容</div>      ✗ 含文本
<div></div>           ✓ 真空
<div> </div>          ✗ 含空格
<div><span></span></div>  ✗ 含子元素
<div></div>           ✓ 真空

苛刻的"空"标准

哪怕只有一个空格、一个换行符，:empty 也认为不是空。但是 HTML 注释不算内容——<div><!-- 注释 --></div> 仍然会被命中。这一行为常用于动态加载场景下，给空容器一个占位样式。

:target 选择器

当 URL 中的锚点 #xxx 指向某个元素时，该元素就是当前 target。点击下方链接亲身体验。

style.css
:target {
  background: 高亮;
  border-left: 4px solid red;
}
index.html
<a href="#s1">第一节</a>
<a href="#s2">第二节</a>
<a href="#s3">第三节</a>

<section id="s1">...</section>
<section id="s2">...</section>
<section id="s3">...</section>

纯 CSS 交互的妙用

仅用 :target 一行 CSS 就能实现"点击高亮"效果，无需任何 JavaScript。这一思路还可扩展为纯 CSS 的标签页切换、模态框开关、单页导航等等，是 CSS 的"伪交互"利器。
