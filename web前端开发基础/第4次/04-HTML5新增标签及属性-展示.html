§ 01 HTML5 网页结构标签 点击区块查看标签说明

HTML5 引入了一系列语义化结构标签，用于替代原来大量使用 <div> 进行布局的方式。语义化标签让代码结构更清晰，有利于 SEO（搜索引擎优化）和无障碍访问。

💡 在 HTML5 之前，开发者习惯用 <div id="header">、<div id="nav"> 等方式定义页面区域。语义化标签的出现让标签名称本身就能表达用途，机器和人都能更轻松地读懂页面结构。

HTML5<header> · 页眉标签
作用
具有引导作用的结构标签，可以包含所有放在页面头部的内容，例如网站 Logo 图像、导航链接、标题、表单等相关内容。
  • —一个文档中可以定义多个 <header> 标签
  • —可以为每个独立内容块单独添加 <header>
  • —也可以用来放置整个页面或某个内容区块的标题
html · header 示例
<header> <h2 align="center">我院通过教育部第二批国家现代学徒制试点单位评审认定</h2> <p align="center"> 发表时间：2017-09-09&nbsp;&nbsp;来源：信息工程系 </p> <hr /> </header>
HTML5<nav> · 导航标签
作用
用于定义导航链接，将具有导航性质的链接归纳在一个区域中，使页面元素的语义更加明确。
💡 并非所有文档都需要使用 <nav>，只用于标注主要的导航链接区域。可根据不同设备控制导航的显示与隐藏，以适应不同屏幕。

可设置的导航类型：网站主导航（页头菜单）、页面内锚点导航（目录）、面包屑导航、侧边栏导航、翻页导航（上一篇/下一篇）。

html · nav 示例
<nav> <ul> <li><a href="#">首页</a></li> <li><a href="#">公司概况</a></li> <li><a href="#">产品展示</a></li> <li><a href="#">联系我们</a></li> </ul> </nav>
HTML5<article> · 独立内容标签
作用
用于定义文档、页面或应用程序中与上下文不相关的独立部分，经常用于定义日志、新闻、用户评论等内容。
  • —标签内容本身必须有意义，且必须独立于文档的其余部分
  • —内部可用多个 <section> 进行进一步划分
  • —适用：论坛帖子、博客文章、新闻故事、评论等
html · article 嵌套 header 与 footer
<article> <header> <h1>秋天的味道</h1> <p>你想不想知道秋天的味道？它是甜、是苦、是涩。</p> </header> <section>文章正文内容……</section> <footer> <p>著作权归 XXXXXX 公司所有。</p> </footer> </article>
🔑 <article> 与 <section> 可以互相嵌套，没有上下级关系。<article> 内可以有多个 <section>，<section> 内也可以有多个 <article>。
HTML5<aside> · 侧边栏标签
作用
用来定义当前页面或文章的附属信息部分，可以包含与主要内容相关的引用、侧边栏、广告、导航条等有别于主要内容的部分。
用法位置说明
用法一置于 <article> 内部用于定义文章主要内容的附属信息
用法二置于 <article> 外部用于定义整个页面或网站的附属信息（如右侧菜单）
html · aside 两种用法
<article> <section>文章主要内容</section> <aside>文章的其他相关信息</aside> <!-- 用法一 --> </article> <aside>右侧菜单</aside> <!-- 用法二：页面级侧边栏 -->
HTML5<section> · 区段标签
作用
用于定义一段专题性的内容，一般会带有标题，主要应用在文章的章节划分中。
  • ✓具有语义化，仅为样式或脚本使用时应改用无语义的 <div>
  • ✓如果 <article>、<aside> 或 <nav> 更符合条件，则无需使用 <section>
  • ✗没有标题的内容模块不要使用 <section>
  • —与 <article> 可以互相嵌套，没有上下级关系
⚠ 常见误区：不要把 <section> 当作通用容器来使用，通用容器请用 <div>。<section> 的核心是"专题性内容 + 通常有标题"。
HTML5<footer> · 页脚标签
作用
用于定义一个页面或区域的底部，可以包含所有放在页面底部的内容，如版权信息、作者信息、联系方式等。

与 <header> 类似，一个文档中可以使用多个 <footer> 元素（例如，整个页面有页脚，每篇文章也可以有自己的页脚）。

html · footer 两种用法
<article> <footer> <p>版权所有 © 2024 · 著作权归原作者所有</p> </footer> </article> <footer> <!-- 整个页面的页脚 --> <p>© 2024 浙江安防职业技术学院 版权所有</p> </footer>
结构标签对照表
标签语义常见用途
<header>页眉文章标题、发布信息、Logo + 导航
<nav>导航网站主菜单、面包屑导航
<article>独立文章博客正文、新闻、论坛帖子
<aside>侧边栏 / 附属广告、相关链接、补充说明
<section>区段文章内的章节划分（带标题）
<footer>页脚版权信息、联系方式
§ 02 HTML5 分组标签
HTML5<figure> 与 <figcaption> · 媒体内容标签
作用
<figure> 用于定义独立的网页内容，例如图像、图表、照片、代码等。<figcaption> 嵌套在 <figure> 中，为其添加标题说明。
  • —一个 <figure> 内最多允许使用一个 <figcaption>
  • —<figcaption> 应置于 <figure> 的第一个或最后一个子元素位置
  • —删除 <figure> 内容，不应对文档流产生影响
html · figure 示例
<figure> <img src="images/zjaf.jpg" alt="校园规划图"> <figcaption>校园规划图</figcaption> </figure>
HTML5<hgroup> · 标题组标签
作用
用于将多个标题（主标题、副标题或子标题）组成一个标题组，通常与 <h1>~<h6> 组合使用。不会改变标题的样式。
  • ✗如果只有一个标题元素，不建议使用 <hgroup>
  • ✓含多个标题标签时，通常将 <hgroup> 放到 <header> 标签中
html · hgroup 示例
<header> <hgroup> <h1>浙江安防职业技术学院</h1> <h2>——培养公共安全管理人才的高职院校</h2> </hgroup> <nav>…</nav> </header>
§ 03 页面交互标签

HTML5 新增的交互标签，无需 JavaScript 即可实现一些基础的用户交互效果。

HTML5<details> 与 <summary> · 展开 / 折叠标签
作用
<details> 描述文档或某部分的细节，内容默认隐藏。<summary> 作为第一个子元素，定义可见的点击标题。用户单击标题时，显示或隐藏其余内容。
实时演示 · 原生 <details> 折叠效果（可点击）
什么是 <details> 标签？
<details> 标签用于描述文档或文档某个部分的细节。<summary> 标签作为 details 的第一个子元素，用于定义 details 的可见标题。当用户单击标题时，会显示或隐藏 details 中的其他内容。

html · details + summary 示例
<details> <summary>details 标签和 summary 标签</summary> details 标签用于描述文档或文档某个部分的细节。 summary 标签作为 details 的第一个子元素，用于定义标题。 标题是可见的，当用户单击标题时，会显示或隐藏其他内容。 </details>
HTML5<progress> · 进度条标签
作用
用于定义运行中的任务进度（进程）。
属性描述
max规定需要完成的值（总量）
value规定进程的当前值；不设置时显示为动态滚动状态
可视化演示 · <progress> 两种用法

动态

html · progress 两种用法
<!-- 用法一：任务正在进行（不确定进度） --> 工作进行中：<progress></progress> <!-- 用法二：表示已完成的任务量 --> 已修满学分：<progress value="45" max="100"></progress>
HTML5<meter> · 度量标签
作用
定义已知范围内的度量值，仅用于有明确最大值和最小值的场景，如磁盘使用、相关性评分、电量显示等。
属性描述
min最小值，默认为 0
max最大值，默认为 1
value当前度量值
low定义"低"值的分界线
high定义"高"值的分界线
optimum定义最佳值，浏览器可能据此附着不同颜色
title鼠标悬停时显示的提示文字

💡
<progress> vs <meter>：
<progress>：表示任务的完成进度（如下载、上传）
<meter>：表示已知范围内的量（如评分、电量、磁盘占用）
§ 04 层次语义标签（文本语义标签）

层次语义标签用于赋予文本内容更精确的语义含义，有助于搜索引擎理解和无障碍访问。

HTML5<time> · 时间标签
作用
以机器可读的方式对日期和时间进行编码。浏览器不会呈现视觉效果，主要供搜索引擎、日历应用识别。
属性描述
datetime定义具体时间（如 15:00）或日期（如 2010-10-10）
pubdate定义发布日期，通常值为 pubdate
效果展示 · <time> 标签渲染

我们每天早上 9:00 开始营业。

我在 情人节 有个约会。

发布于 2017-10-27

html · time 示例
<p>我们每天早上 <time>9:00</time> 开始营业。</p> <p>我在 <time datetime="2018-02-14">情人节</time> 有个约会。</p> <p>发布于 <time datetime="2017-10-27" pubdate="pubdate">2017-10-27</time></p>
HTML5<mark> · 高亮标签
作用
定义带有记号的文本，实现高亮显示某些字符，以引起用户注意（类似荧光笔效果）。
效果展示 · <mark> 高亮效果
燕子去了，有再来的时候；杨柳枯了，有再青的时候；桃花谢了，有再开的时候。 但是，聪明的，你告诉我，我们的日子为什么一去不复返呢？ ——是有人偷了他们罢……从水盆里，从水盆里过去；吃饭的时候， 日子从饭碗里过去；默默时，便从凝然的双眼前过去。
html · mark 示例
<mark>一去不复返</mark>呢？ 从水盆里，<mark>从水盆里过去</mark>； 日子<mark>从饭碗里过去</mark>；
HTML5<cite> · 作品引用标签
作用
定义作品的标题（书籍、歌曲、电影、电视节目、绘画、雕塑等）。<cite> 标签定义的文本将以斜体显示。
⚠ 人名不属于作品的标题，不应使用 <cite>。
html · cite 示例
<p>山不在高，有仙则名。水不在深，有龙则灵。</p> <cite>——《陋室铭》</cite>
§ 05 HTML5 全局属性

全局属性是可以应用于任何 HTML 元素的属性，HTML5 新增了以下几个常用全局属性。

全局属性draggable · 拖动属性
作用
规定元素是否可以被拖动。
属性值效果
false默认值，不可拖动
true选中后可拖动
html · draggable 示例
<p draggable="true"> draggable 只是开启拖动能力，实际拖放逻辑需配合 JavaScript 的 drag 事件实现 </p> <img src="images/wlw.png" width="460" draggable="true">
⚠draggable="true" 只是开启拖动能力，实际的拖放交互逻辑需要配合 JavaScript 的 drag 事件实现。
全局属性hidden · 隐藏属性
作用
规定元素是否可见。设置后元素被隐藏，且不占据页面空间。
设置效果
不设置元素正常显示
hidden元素被隐藏（不占据页面空间）
html · hidden 示例
<img src="images/wlw.png" width="460" hidden>
💡hidden 与 CSS 的 visibility: hidden 的区别：hidden 属性使元素不占据空间，而 visibility: hidden 仍然占据空间。
全局属性spellcheck · 拼写检查属性
作用
规定是否对元素的输入内容进行拼写和语法检查，主要用于文本输入框。
属性值效果
true默认值，检测拼写和语法
false不检测
html · spellcheck 示例
开启拼写检查： <textarea cols="40" rows="3" spellcheck="true"></textarea> 关闭拼写检查： <textarea cols="40" rows="3" spellcheck="false"></textarea>
全局属性contenteditable · 内容可编辑属性
作用
规定元素的内容是否可以由用户编辑。元素必须可以获得鼠标焦点，且不是只读的。
实时演示 · 点击下方列表可直接编辑内容

<!-- 不可编辑 --> <ul> <li>菜单1</li> </ul> <!-- 可编辑 --> <ul contenteditable="true"> <li>菜单1</li> </ul>

选用标签的决策思路
需要定义页面区域？
├── 有导航链接
└── <nav>
├── 是头部内容
└── <header>
├── 是尾部内容
└── <footer>
├── 是独立完整内容（可被转载）
└── <article>
├── 是带标题的专题区段
└── <section>
├── 是附属 / 侧边内容
└── <aside>
└── 仅为布局分组，无语义
└── <div>
需要为文字添加样式？
├── 要表达重要性
└── <strong>（不要只用 <b>）
├── 要表达强调
└── <em>（不要只用 <i>）
├── 要引用作品名
└── <cite>
├── 要标注时间日期
└── <time>
└── 要高亮关键词
└── <mark>
📌
学习建议：按照"理解语义 → 查看示例 → 动手实践 → 综合运用"的路径进行学习。每个标签的核心是理解它的语义含义，而不只是记忆其视觉效果。多动手尝试将各标签组合使用，才能真正掌握 HTML5 的语义化网页结构设计。
