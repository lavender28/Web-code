§ 01 标签关系 Father and son or brothers

HTML 标签之间存在两种基本关系：嵌套关系（父子关系）和并列关系（兄弟关系）。理解这两种关系是读懂 HTML 结构、解决布局 bug 的基础。

交互演示 · 点击标签查看关系说明
<body> ← 父元素
<h1>文章标题</h1> ← body 的子元素；与 <p>、<ul> 是兄弟关系
<p>段落内容</p>
<ul> ← ul 与 li 是父子关系
<li>列表项一</li> ← 兄弟关系
<li>列表项二</li>
</ul>
</body>
嵌套关系（父子关系）
一个标签包含在另一个标签内部。外层称为父元素，内层称为子元素。

示例：<ul> 是 <li> 的父元素
并列关系（兄弟关系）
两个标签处于同一层级，有共同的父元素。

示例：<h1> 和 <p> 都是 <body> 的子元素，它们互为兄弟
⚠ 实际开发中多层嵌套最容易出问题。可利用编辑器的代码折叠、缩进高亮，或 AI 工具帮助理清标签的层次关系。
练手思考
💡
思考题：
1. 对于 <li> 标签来说，谁是它的直接父元素？
2. <h1> 和 <ul> 之间是什么关系？
§ 02 标签属性

属性（Attribute）是写在 HTML 标签开始标签内的附加信息，用来为标签提供额外的配置或描述。属性以键值对的形式书写。

html · 属性语法格式
<!-- 语法格式：属性名="属性值" --> <标签名 属性名="属性值">内容</标签名> <!-- 示例：为 h3 设置居中对齐 --> <h3 align="center">静夜思</h3> <!-- 多个属性之间用空格分隔 --> <hr align="center" width="100px" color="red">
  • ①
    属性写在开始标签内，标签名后面
  • ②
    属性名与属性值之间用等号 = 分隔
  • ③
    属性值用双引号包裹（也可以是单引号，统一即可）
  • ④
    多个属性之间用空格分隔，顺序一般无关紧要
§ 03 标题标签 h1 ~ h6 点击标题行查看字号

在新闻和文章页面中，都离不开标题来突出显示文章主题。HTML 提供了六级标题标签，从 <h1>（最重要）到 <h6>（最次要），标题文字会加粗显示，且每行只显示一个（块级元素）。

交互演示 · 点击各级标题查看视觉权重

一级标题 2rem / bold
h2
二级标题 1.5rem / bold
h3
三级标题 1.17rem / bold
h4
四级标题 1rem / bold
h5
五级标题 0.83rem / bold
h6
六级标题 0.67rem / bold
点击任意一行标题，查看其语义说明
html · 标题标签语法
<h1>一级标题（最重要）</h1> <h2>二级标题</h2> <h3>三级标题</h3> <!-- h4、h5、h6 以此类推 -->
使用规范
  • ★
    h1 唯一性：每个页面最好只使用一次 <h1>，用于页面的顶级标题，如新闻标题、网页 Logo 区域
  • —
    层次性：在现有的六个层次中，除非有必要，否则每页使用不超过三个层级，避免层级混乱
§ 04 段落标签 p
定义
<p> 元素表示文本的一个段落（Paragraph）。每行只显示一个，文字显示不开会自动换行。段落的样式（字体、行距、颜色等）请用 CSS 设置。
html · 段落标签语法
<p>这是第一段文字内容，文字过多时会自动换行。</p> <p>这是第二段文字内容，段落之间有默认间距。</p> <!-- 强制换行使用 <br> --> <p> 床前明月光，<br> 疑是地上霜。<br> </p>
为什么需要结构化（语义化）？
  • ✓
    更清晰的代码结构：开发者和维护者能快速理解页面各区域的用途
  • ✓
    对搜索引擎更友好：爬虫可以准确识别标题、段落等内容的权重，有助于 SEO
  • ✓
    更好的可访问性：屏幕阅读器依赖语义标签帮助视障用户理解网页内容
综合实践：静夜思

§ 05 块级元素与内联元素 Kuai or n

在 HTML 中，所有元素按显示方式分为两大类：块级元素和内联元素。分类不同决定了展示形式不一样。

可视化对比 · 块级 vs 内联元素布局
块级元素（block）— 独占一行：
<p> 段落（独占整行）
<h2> 二级标题（独占整行）
<div> 容器（独占整行）
内联元素（inline）— 可以一行多个：
<strong> 加粗 <em> 斜体 <a> 链接 <span> 通用
块级元素
• 独占一行
• 可以嵌套其他元素
• 常见：p、h1~h6、div、ul、ol
内联元素
• 可以一行放多个
• 不能嵌套块级元素
• 常见：strong、em、a、span
⚠
注意事项：
• <p> 标签里面不要放其他块级元素，里面主要放文字相关内容（内联元素）
• 元素分类不是不可更改的，CSS 中的 display 属性可以改变元素的显示模式

§ 07 注释标签 not see~

注释标签用于在 HTML 代码中添加不会在浏览器中显示的说明或备注。注释的内容只在源代码中可见，对页面展示没有任何影响。

对比演示 · 注释内容在页面上不可见
这段文字正常显示在网页上。
👁‍🗨 注释内容：<!-- 这里是注释，用户在浏览器中看不到这行内容 -->
此行仅在源码中可见，不会渲染到页面。
html · 注释语法
<!-- 这里是单行注释 --> <!-- 这里是 多行注释 可以跨越多行 --> <p>这段文字正常显示。</p> <!-- <p>这段被注释掉，不会显示。</p> --> <p>下一段文字。</p>
💡
快捷键： Ctrl + /（Windows）或 Cmd + /（Mac）可以在 VS Code 中快速为选中代码添加或取消注释。

三大用途：
① 代码说明：解释某段代码的功能或注意事项
② 临时禁用：调试时临时屏蔽某段代码而不删除
③ 团队协作：给其他开发者留下说明信息
§ 08 无序列表 ul 顺序无关紧要的列表

无序列表用于展示一组顺序无关紧要的项目。在实际开发中，无序列表非常常用，常用于导航菜单、商品列表、整齐对齐的模块等布局场景。

实时演示 · 无序列表
  • 猪爸爸
  • 猪妈妈
  • 佩奇
  • 乔治
html · ul 无序列表
<ul> <li>猪爸爸</li> <li>猪妈妈</li> <li>佩奇</li> <li>乔治</li> </ul>
⚠
注意事项：
• <ul> 里面只能包含 <li> 元素，不能直接放其他标签
• <li> 里面可以放任何其他 HTML 元素（文字、图片、链接等）
• 无序列表在开发布局中非常重要，是最常用的列表类型
§ 09 有序列表 ol 顺序有关的列表

有序列表用于展示一组有先后顺序关系的项目，浏览器会自动在每个 <li> 前加上编号。在实际开发中有序列表较少使用，即使有顺序，开发者一般也会用其他方式代替。

实时演示 · 有序列表
  1. 看视频
  2. 写代码
  3. 做笔记
  4. 多复习
html · ol 有序列表
<ol> <li>看视频</li> <li>写代码</li> <li>做笔记</li> <li>多复习</li> </ol>
💡
<ol> 与 <ul> 结构完全相同，只是容器标签不同。
同样：<ol> 里面只能包含 <li> 元素；<li> 里面可以放任何其他元素。
§ 10 描述列表 dl 标记一组项目及相关描述

描述列表用于标记一组项目及其相关描述，常见于网页底部导航栏，对分类项目进行说明展示。它由三个标签共同构成。

实时演示 · 描述列表（常见于页面底部）
家电
电视
冰箱
空调
烟灶
html · dl 描述列表
<dl> <dt>家电</dt> <!-- 术语/标题，一个 dt 可对应多个 dd --> <dd>电视</dd> <!-- 描述/定义，通常缩进显示 --> <dd>冰箱</dd> <dd>空调</dd> <dd>烟灶</dd> </dl>
dl / dt / dd 三个标签说明
标签作用注意事项
<dl>定义列表的容器只能包含 <dt> 和 <dd> 元素
<dt>定义被描述的术语/标题通常显示为左对齐或加粗；一个 dt 可以对应多个 dd
<dd>包含术语的定义或描述通常缩进显示；可以包含段落、图片、链接等其他元素
⚠
注意事项：
• <dl> 里面只能包含 <dt> 和 <dd> 元素
• 一个 <dt> 可以对应多个 <dd>
• <dd> 可以包含段落、图片、链接等其他 HTML 元素

📌
知识总结：
① 标题共六级（h1~h6），h1 每页用一次，建议不超过三级
② 语义化好处：代码结构清晰、SEO 友好、可访问性好
③ 优先使用有语义的文本标签（strong / em / ins / del）
④ 三种列表：无序列表最重要，描述列表其次，有序列表了解即可
⑤ ul / ol 只能包含 li；dl 只能包含 dt 和 dd
