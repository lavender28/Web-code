CSS3 链接伪类选择器

伪类选择器（pseudo-class）用于根据元素的特殊状态来选中元素。链接伪类专门针对 <a> 元素的不同交互状态，是 CSS 中最常用的一组伪类。

五大核心伪类
:link未访问链接
匹配所有尚未被用户访问过的 <a> 元素。元素必须有 href 属性才会被匹配。
浏览器通常默认将其渲染为蓝色带下划线。
:visited已访问链接
匹配用户已经访问过的链接。浏览器记录在历史记录中的 URL 均被视为已访问。
出于隐私保护，只允许修改 color、background-color 等少数属性。
:hover鼠标悬停
当用户将鼠标指针悬停在元素上方时触发。不限于链接，适用于任何元素。
触屏设备上 :hover 行为不稳定，需额外考虑。
:active激活状态
在用户点击链接的瞬间（鼠标按下到抬起之间）匹配元素，时间极短。
常用于制作按压反馈效果（如按钮下沉动画）。
:focus获得焦点
当元素通过键盘 Tab 或点击获得焦点时匹配。是键盘导航可访问性的关键伪类。
不要设置 outline: none，这会严重影响无障碍访问。

LVHA 顺序规则

声明顺序至关重要

CSS 伪类遵循层叠规则，相同优先级下后面的规则覆盖前面的。必须按照 L → V → H → A 的顺序书写，否则某些状态将永远不会显示。

代码示例
/* 标准 LVHA 顺序 */
a:link {
  color: #534AB7;
  text-decoration: underline;
}

a:visited {
  color: #888780;
}

a:hover {
  color: #D85A30;
  text-decoration: none;
}

a:active {
  color: #993C1D;
}
/* 进阶：下划线动画效果 */
a {
  color: #534AB7;
  text-decoration: none;
  position: relative;
  padding-bottom: 2px;
}

a::after {
  content: '';
  position: absolute;
  bottom: 0; left: 0;
  width: 0;
  height: 2px;
  background: currentColor;
  transition: width 0.25s ease;
}

a:hover::after,
a:focus::after {
  width: 100%;
}

a:focus {
  outline: 2px solid #378ADD;
  outline-offset: 3px;
  border-radius: 2px;
}
/* 导航栏链接 — 完整状态管理 */
.nav-link {
  display: inline-flex;
  align-items: center;
  padding: 6px 16px;
  color: #5F5E5A;
  text-decoration: none;
  border-radius: 6px;
  transition: all 0.15s ease;
}

.nav-link:link,
.nav-link:visited {
  color: #5F5E5A;
}

.nav-link:hover {
  color: #534AB7;
  background: #EEEDFE;
}

.nav-link:active {
  transform: scale(0.97);
}

.nav-link:focus-visible {
  outline: 2px solid #7F77DD;
  outline-offset: 2px;
}
快速参考表
选择器触发时机适用元素常见用途
:link链接从未被访问仅 <a href>设置链接默认外观
:visited链接已在历史记录中仅 <a href>区分已读/未读状态
:hover鼠标指针悬浮其上所有元素悬停高亮、展开菜单
:active鼠标按下到释放期间所有元素按压反馈、点击动画
:focus元素获得输入焦点可聚焦元素键盘导航高亮
:focus-visible仅键盘导航聚焦时可聚焦元素仅为键盘用户显示轮廓
:focus-within自身或后代获得焦点容器元素表单组整体高亮
常见注意事项
顺序错误
若将 :hover 写在 :visited 之前，已访问链接的悬停效果将失效，始终显示 :visited 样式。
:visited 限制
浏览器出于隐私保护，:visited 只能修改颜色相关属性，无法改变尺寸、位置或添加背景图。
:focus-visible 推荐
用 :focus-visible 代替 :focus 来显示轮廓，可避免鼠标点击时出现不美观的焦点框。
触屏设备
移动端没有真正的 hover 状态，:hover 可能在点击后保持，需用媒体查询 @media (hover: hover) 隔离。
组合使用
伪类可以链式组合，如 a:visited:hover { } 单独处理"已访问且悬停"的样式，更加精细。
不限于链接
:hover、:active、:focus 适用于任何 HTML 元素，:link 和 :visited 仅限于带 href 的 <a> 元素。
