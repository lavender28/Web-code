图片与超链接

掌握 <img> 标签与 <a> 标签的完整用法，理解路径与链接的本质
<img> 图片标签 路径：绝对 · 相对 <a> 超链接 · 五种形态
§ 01 <img> 标签与核心属性 点击属性名查看说明

在 HTML5 中，插入图片的核心标签是 <img>。它是一个单标签（Self-closing），不需要 </img> 来结尾。<img> 默认必须写两个属性：src 和 alt。

交互演示 · 点击属性名查看详解
<img src="./1.png" alt="王者荣耀截图">
点击上方蓝色属性名，查看该属性的详细说明
两个必须属性
src 属性（Source）
指向要插入的图片地址。告诉浏览器去哪里找这张图，可以是本地图片，也可以是网络链接。
alt 属性（Alternative text）
备选文本。当图片因网络慢、加载失败或被屏蔽时，会显示这段文字。
• 无障碍友好：屏幕阅读器会朗读这段文字，帮助视障用户理解图片内容。
• SEO 友好：搜索引擎（百度、Google）通过它来理解图片主题。
html · 基础写法
<img src="./1.png" alt="这是一张王者荣耀的图片">
⚠
属性书写规则（必须记住）：
① 键值对形式：属性="值"
② 属性之间必须用空格分开，顺序无所谓
§ 02 图片的其他属性

除了 src 和 alt 这两个"必选项"，<img> 标签还提供了几个实用的属性，用来控制图片的外观、行为和性能。

属性作用示例
width设置图片的宽度（单位像素，不需要写 px）width="300"
height设置图片的高度height="200"
title鼠标悬停在图片上时显示的提示文字（工具提示）title="点击查看原图"
loadingHTML5 新特性。常用 lazy（懒加载），图片滚动到视口附近时才加载，提高页面打开速度loading="lazy"
⚠
宽高设置的小细节：
只设置 width 或 height 其中一个，浏览器会根据原始比例自动计算另一个。
若同时设置两个不符合比例的数值，图片会被拉伸或挤压变形。
强烈建议用 CSS 控制尺寸，更灵活，不容易变形。
html · 完整属性示例
<img src="./photo.jpg" alt="校园风景照片" width="300" title="点击查看原图" loading="lazy" >
§ 03 常见图片格式

选择正确的图像格式非常重要，它直接影响页面的加载速度（性能）和视觉质量。HTML5 支持多种图像格式，分为位图（由像素组成）和矢量图（由数学公式组成）。

📸
JPEG / JPG
位图 · 有损压缩
压缩率高，文件小，但不支持透明
💎
PNG
位图 · 无损压缩
支持透明背景，文件较大。适合 Logo
🎞️
GIF
位图 · 动画
支持简单动画，色彩仅 256 色
🚀
WebP
现代推荐 · Google 出品
比 JPEG 小 30%，支持透明和动画，现代网页首选
📐
SVG
矢量图 · 代码描述
无限放大不失真，文件极小。适合图标
⚡
AVIF
新兴格式 · 超高压缩
压缩率优于 WebP，浏览器支持率快速增长
开发选用建议
选择图片格式时，问自己：
├── 追求页面极致速度？
└── WebP / AVIF，兼容降级用 JPEG/PNG
├── 需要透明背景（Logo、贴图）？
└── PNG（静态）或 WebP（支持动画）
├── 需要动画效果？
└── WebP > GIF（WebP 动画体积更小）
├── 图标 / 扁平插图，需要任意缩放？
└── SVG（无限放大不失真）
└── 复杂摄影照片，无特殊需求？
└── JPEG（压缩率高，体积小）
§ 04 路径：绝对路径与相对路径

路径（Path）就像给浏览器的一张"地图"，告诉它去哪里寻找并加载图片文件。如果路径写错了，网页上就会出现"碎图"图标。HTML 中主要使用两种路径方式。

路径类型绝对路径（Absolute Path）
定义
图片在网络上的完整地址（通常以 http:// 或 https:// 开头），或者电脑硬盘上的完整位置。
  • ✓
    优点：无论 HTML 文件保存在哪里，只要有网，图片就能显示。
  • ✗
    缺点：如果对方网站删除了图片，链接就会失效。
html · 绝对路径示例
<img src="https://example.com/images/logo.png" alt="网站 Logo">
路径类型相对路径（Relative Path）
定义
相对于当前 HTML 文件所在位置来寻找图片。这是开发中最常用的方式。
交互演示 · 点击查看不同路径场景
📁 project/
📄 index.html ← 当前文件
🖼 pic.jpg ← 目标图片
src="pic.jpg" 或 src="./pic.jpg" — 图片与 HTML 在同一文件夹
符号含义示例
./ 或直接文件名图片和 HTML 在同一个文件夹里src="pic.jpg"
src="./pic.jpg"
文件夹名/图片在当前目录的下一层文件夹里src="images/pic.jpg"
../跳出当前文件夹，去上一层寻找src="../pic.jpg"
✓ 小技巧：在 VS Code 里输入 ./ 会智能提示文件夹和文件，超级方便！
§ 05 <a> 标签基础与 href 属性

超链接是网页的灵魂，它让独立的页面连接成网。HTML 使用 <a>（Anchor，锚点）标签来创建连接。它是一个双标签，用户点击标签之间的内容（文字或图片），就会触发跳转。

href 属性（Hypertext Reference）
超文本引用。就像一张车票，告诉浏览器目的地的地址。没有 href，链接就无法跳转。
html · a 标签基础写法
<a href="https://www.baidu.com">百度一下</a>
  • —
    https://www.baidu.com 是目标地址（外部绝对路径）
  • —
    "百度一下"是用户在网页上看到并点击的蓝色文字
  • —
    链接内容不仅可以是文字，也可以是图片（将 <img> 放在 <a> 内部即可）
§ 06 target 属性 · 控制打开方式

默认情况下，点击链接时，新页面会在当前标签页中打开，覆盖掉原本正在浏览的网页。target 属性用于控制链接的打开位置。

属性值效果使用场景
_self在当前标签页打开（默认值）站内页面跳转
_blank在新标签页打开外部链接、保留当前页面
_parent在父框架中打开iframe 场景
_top在整个窗口中打开iframe 场景
html · target 示例
<!-- 默认：在当前标签页打开 --> <a href="https://www.baidu.com">百度一下</a> <!-- 在新标签页打开，保留当前页 --> <a href="https://www.baidu.com" target="_blank">百度一下（新窗口）</a>
§ 07 空链接（占位链接）

有时链接的目的地还没确定，或者只想配合 JavaScript 制作点击特效。这时需要用到空链接。

交互演示 · 两种写法对比
写法 A · href="#"
点击后页面会瞬间跳回最顶部！这是新手常见的"小陷阱"——当页面已经滚动到下方时，点击 # 链接会意外回滚到顶部。
html
<a href="#">点击这里</a>
# — 点击后页面会滚回顶部，适合临时占位
§ 08 下载链接

默认情况下，如果链接指向图片或 PDF，浏览器通常会直接在网页里打开预览。如果希望用户点击后直接将文件保存到本地，HTML5 提供了 download 属性。

html · download 两种写法
<!-- 基础下载：加上 download 即可触发下载 --> <a href="game.zip" download>下载游戏压缩包</a> <!-- 下载并重命名：给下载文件指定更清晰的名字 --> <a href="report_2026_v3_final_edit.pdf" download="年度报告.pdf">下载报告</a>
  • ✓
    只需加上 download 这个词即可触发下载（基础用法）
  • ✓
    download="文件名" 可以给下载文件重命名，不管服务器上原来的名字多乱
  • ✗
    跨域链接（外部网站文件）的 download 属性可能不生效，受浏览器安全策略限制
§ 09 邮件链接

想让用户一键给你发邮件？不需要手动复制邮箱地址。只需在 href 里使用 mailto: 协议，用户点击后，设备上的默认邮件客户端就会自动弹出，收件人已填好。

html · 邮件链接写法
<!-- 基础：点击打开邮件客户端并填入收件人 --> <a href="mailto:hello@example.com">给我写信</a> <!-- 进阶：同时预填主题和正文内容 --> <a href="mailto:hello@example.com?subject=合作咨询&body=你好，我想了解……"> 发送合作邮件 </a>
💡 mailto: 链接会调用用户本机已安装的邮件客户端（如 Outlook、Apple Mail）。若用户未安装邮件客户端，此链接可能无效，建议同时提供可见的邮箱文本作为备选。
§ 10 锚点链接（页内跳转） 下方有实时演示

锚点链接就像网页内部的"传送门"，允许用户点击后瞬间跳跃到当前页面的特定位置。在长篇文章的"目录"或"回到顶部"功能中非常常见。

实现两步走
第一步：设定目标 — 给目标标签添加唯一的 id 属性（就像身份证号，同页面不能重复）
第二步：创建链接 — href 以 # 开头，紧跟 id 名称（# 在这里的意思是"在当前页面中寻找"）
html · 锚点链接完整示例
<!-- 第一步：目录链接（点击可跳转） --> <a href="#history">点击阅读品牌历史</a> <a href="#products">查看产品介绍</a> <!-- 第二步：目标位置（带有 id 的标签） --> <h2 id="history">品牌历史</h2> <p>创立于 2000 年……</p> <h2 id="products">产品介绍</h2> <p>我们的核心产品……</p>

学习建议：图片和链接是网页中最高频的两类元素，建议多动手练习不同路径写法和链接形态。理解"路径"是学习 HTML 的核心思维之一——在项目结构变复杂后，路径写错是最常见的 bug 来源，养成用 ./ 开头的相对路径习惯，搭配 VS Code 智能提示，可以大幅减少错误。
