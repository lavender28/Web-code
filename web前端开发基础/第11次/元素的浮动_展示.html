§ 01 元素的类型 点击卡片展开详情

HTML 标签按照显示方式被划分为三种类型。理解各类型的特点，是学习浮动与布局的基础前提。

行内元素（inline）
  • ✗ 不独占一行，与相邻行内元素排在同一行
  • ✗ 完全不支持盒子模型，无法设置宽度、高度、对齐
  • ✓ 宽高由内容本身决定，内容多宽就占多宽
常见标签：
<span><a> <strong><em> <i><u> <s><del>
块元素（block）
  • ✓ 独占一行或多行，默认宽度 100%
  • ✓ 完全支持盒子模型，可自由设置宽高、内外边距
  • ✓ 可包含行内元素和其他块元素
常见标签：
<div><h1~h6> <p><ul> <ol><li> <header><footer>
行内块元素（inline-block）
  • ✗ 不独占一行（像行内元素）
  • ✓ 可设置宽高、对齐等属性（像块元素）
  • △ 支持部分盒子模型
常见标签：
<img><input>

§ 02 元素类型的转换 切换按钮实时预览

实际开发中，有时需要改变元素的默认类型：例如让 <a> 标签可以设置宽高，或让 <div> 不独占一行。可使用 display 属性进行转换。

css
选择器 { display: inline | block | inline-block | none; }
属性值效果典型应用场景
inline转换为行内元素，不可设宽高让 <div> 不独占一行
block转换为块元素，独占一行让 <a> 可以设置宽高
inline-block行内排列且可设宽高导航菜单项横向排列
none隐藏元素，不占页面空间切换显示与隐藏状态
交互演示 · 切换 display 值

inline — 三个元素在同一行排列，无法设置宽高
⚠ 行内元素只可以定义左右外边距，上下边距设置无效。需要上下间距时，应先将其转换为块或行内块元素。
§ 03 overflow 属性 控制内容溢出的显示方式
定义
当盒子内的内容超出盒子设定的大小时，就会产生溢出现象。overflow 属性用于规定溢出内容的显示状态。
css
选择器 { overflow: visible | hidden | scroll | auto; }
属性值效果适用场景
visible内容显示在盒子框外（默认值）不处理溢出
hidden超出内容被隐藏裁剪图片裁切、清除浮动
scroll始终显示滚动条固定高度的内容区域
auto有溢出才出现滚动条内容不确定的区域

§ 04 元素的浮动 改变文档流，实现横向排列
定义
浮动：改变普通文档流的排列方式，使块元素可以在同一行中排列，让网页布局更加灵活方便。设置了浮动属性的元素将脱离标准文档流的控制，移动到父元素内指定方向的位置。
css
选择器 { float: none | left | right; }
属性值效果
none默认值，元素不浮动，按文档流排列
left元素向左浮动，脱离文档流
right元素向右浮动，脱离文档流

浮动的四条核心原则
01有浮动属性的元素一定会移动，但未必改变原来的位置▾
若父容器足够宽，浮动元素视觉上可能仍在原位，但已脱离文档流，后续元素排布已受影响。
02有浮动属性的元素会脱离原有文档流▾
脱离文档流后，元素不再占据原来的位置，后面的非浮动元素会"填上来"取代其位置，但文字内容会环绕浮动元素显示。
03若上一行无浮动元素，下一行浮动元素无法上移▾
浮动元素只能在同一行内横向移动，遇到上方无浮动元素时，无法跨行向上浮动。
04非浮动元素会忽略前面的浮动元素并取代其位置▾
非浮动元素的盒子边框会延伸到浮动元素下方，产生"文字环绕"效果。这也是造成父元素高度塌陷的根本原因。
css · 示例：三个 div 左浮动
#box1 { background-color: red; float: left; }
#box2 { background-color: blue; float: left; }
#box3 { background-color: green; float: left; }
§ 05 清除浮动 避免浮动对其他元素的影响

浮动元素脱离文档流后，不再占据原来的位置，会对后面相邻的固定元素造成位置上的影响。为避免浮动对其他元素的干扰，需要清除浮动。

css
选择器 { clear: left | right | both; }
属性值效果
left清除左浮动的影响
right清除右浮动的影响
both同时清除左右两侧浮动影响（最常用）
⚠ clear 只能清除兄弟元素之间的浮动影响。当父元素内的子元素全部浮动时，父元素高度变为 0（高度塌陷），clear 属性无法直接解决，需用下列专门方法。

问题：高度塌陷
子元素全部浮动后，父元素高度变为 0，无法正常包裹内容。
两种清除方法说明
方法一：空标记法
在浮动子元素后面添加空的块元素，并设置 clear: left，父元素高度恢复正常。
✓ 效果直观   ✗ 增加无意义标签
方法二：overflow: hidden
对父元素设置 overflow: hidden，触发 BFC，自动包含浮动子元素。
✓ 代码简洁   ✗ 子元素超出会被裁剪
html + css · 方法一：空标记
.box4 { clear: left; } /* 空标记样式 */ <div class="father"> <div class="box1">box1</div> <div class="box2">box2</div> <div class="box3">box3</div> <div class="box4"></div> <!-- 空标记 --> </div>
css · 方法二：overflow: hidden
.father { background-color: #c0c0c0; overflow: hidden; /* 关键：加在父元素上，触发 BFC */ }
