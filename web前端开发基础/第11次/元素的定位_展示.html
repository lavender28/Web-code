元素的定位

掌握四种定位模式，实现元素的精确控制与布局
§ 01 元素的定位概述

浮动布局虽然灵活，但无法对元素位置进行精确控制。CSS 的 position 属性可以实现元素在页面上的精准放置，是网页布局的核心技术之一。

元素的定位由两个部分共同决定，二者缺一不可：

（1）定位模式

通过 position 属性设置，决定元素以谁为参照基准进行定位：

css · 语法格式
选择器 { position: static | relative | absolute | fixed; }
属性值定位模式是否脱离文档流
static静态定位（默认）✗ 否
relative相对定位✗ 否（占位保留）
absolute绝对定位✓ 是
fixed固定定位✓ 是
（2）边偏移

配合定位模式，通过以下四个属性精确指定元素的具体坐标（静态定位除外）：

属性说明
left左侧偏移量，定义元素距参照物左边线的距离
right右侧偏移量，定义元素距参照物右边线的距离
top顶端偏移量，定义元素距参照物上边线的距离
bottom底部偏移量，定义元素距参照物下边线的距离
💡 定位模式决定"以谁为参照"，边偏移决定"偏移多少"。单独设置任一个均无法达到预期效果，必须结合使用。
§ 02 静态定位（static）
定义
静态定位是网页元素默认的定位方式，元素按照标准文档流正常排列，不受边偏移属性控制。
css
选择器 { position: static; }
  • ✓所有元素未设置 position 时均默认为静态定位
  • ✓静态定位元素不能通过 left、right、top、bottom 改变位置
  • ✓实际开发中很少主动声明，主要用于重置其他定位模式
⚠ 常见误区：没有设置 position 属性，不代表元素没有位置。它依然以静态定位显示在文档流中的默认位置。
§ 03 相对定位（relative） 点击按钮查看偏移效果
定义
网页元素相对于其在原文档流中的位置进行定位偏移，偏移后原位置仍然保留，其他元素不会填补该空白。
css
选择器 { position: relative; }
  • ✓元素相对于自身原始位置进行偏移
  • ✓不脱离文档流：原位置依然保留占位，后续元素不会上移填补
  • ✓可通过 top、left、bottom、right 设置偏移量

html + css · 相对定位示例
.father { width: 300px; height: 300px; background-color: #c0c0c0; } .box2 { position: relative; /* 相对定位 */ left: 150px; /* 相对原位置向右偏移 150px */ top: 100px; /* 相对原位置向下偏移 100px */ }
💡 效果说明：box2 向右移动 150px、向下移动 100px，但 box2 原来的位置仍保留空白区域，box3 不会上移填补。这是相对定位与绝对定位的本质区别。
✓
常见用途：①对元素进行微调而不影响其他元素布局；②作为绝对定位子元素的参照父容器（即"子绝父相"中的"父相"部分，详见 §04）。
§ 04 绝对定位（absolute） 点击切换参照对象
定义
将元素相对于距离其最近的、已设置定位的祖先元素进行定位；若所有祖先元素均未设置定位，则依据 body 根元素（浏览器窗口）进行定位。
css
选择器 { position: absolute; }
  • ✓脱离文档流：原位置不再保留，其他元素会填补其空缺
  • ✓参照对象为最近的已定位祖先元素（position 为 relative / absolute / fixed 的祖先）
  • ✓若所有祖先均为静态定位，则以 body（浏览器窗口）为参照

核心口诀：子绝父相

+

子元素使用绝对定位（absolute），父元素使用相对定位（relative）。
父元素设为 relative 后不脱离文档流（保留自身占位空间），
同时为绝对定位的子元素提供精准的参照基点。
css · 示例一：父元素无定位，以 body 为参照
.father { width: 300px; height: 300px; /* 注意：父元素未设置 position，视为 static */ } .box2 { position: absolute; /* 绝对定位 */ left: 150px; /* 距浏览器窗口左边线 150px */ top: 100px; /* 距浏览器窗口顶部边线 100px */ }
css · 示例二：子绝父相（推荐做法）
.father { position: relative; /* 父元素设为相对定位，作为参照物 */ width: 300px; height: 300px; } .box2 { position: absolute; /* 绝对定位 */ left: 150px; /* 距父元素左边线 150px */ top: 100px; /* 距父元素顶部边线 100px */ }
⚠
注意事项：
① 仅设置绝对定位而不设偏移量，元素位置不变，但已脱离文档流，可能与其他元素重叠。
② left 与 right 冲突时，以 left 为准；top 与 bottom 冲突时，以 top 为准。
§ 05 固定定位（fixed）
定义
相对于浏览器窗口进行定位，无论页面内容如何滚动，元素始终固定在窗口的指定位置显示。
css
选择器 { position: fixed; }
  • ✓脱离文档流：原位置不再保留，其他元素会填补其空缺
  • ✓参照物始终为浏览器窗口，与任何父元素的定位无关
  • ✓无论滚动条如何拖动、窗口如何缩放，固定定位元素始终显示在指定位置

模拟浏览器视口（可滚动）
position: fixed · 固定导航栏（始终可见）
固定定位效果
向下滚动内容区域，红色导航栏始终停留在顶部。这正是 position: fixed 的典型表现。
css · 典型应用示例
/* 固定在顶部的导航栏 */ .nav-bar { position: fixed; top: 0; left: 0; width: 100%; height: 60px; background: #fff; z-index: 999; } /* 固定在右下角的返回顶部按钮 */ .back-to-top { position: fixed; bottom: 30px; right: 30px; width: 50px; height: 50px; background: #c0392b; color: #fff; }

§ 06 z-index 层叠等级属性 拖动滑块调整层叠顺序
定义
当多个定位元素发生重叠时，z-index 属性用于控制元素的堆叠顺序。取值为正整数、0 或负整数，数值越大，元素越靠上显示。
css
选择器 { z-index: 数值; } /* 正整数、0 或负整数，默认为 0 */
  • ✓取值：正整数、0、负整数，默认值为 0
  • ✓数值越大，元素越靠上（越显示在前面）
  • ✓数值相同时，后写的元素在上面
  • ✓z-index 只对已设置定位（non-static）的元素有效

css · z-index 示例
.box1 { position: absolute; z-index: 1; /* 在下层 */ } .box2 { position: absolute; z-index: 10; /* 在上层，显示在 box1 上方 */ } .box3 { position: absolute; z-index: -1; /* 在最下层，显示在其他元素后面 */ }
总结 四种定位模式对比
定位模式 是否脱离文档流 参照对象 边偏移有效 典型应用
static
静态		 ✗ 否 — ✗ 无效 默认状态，一般不主动设置
relative
相对		 ✗ 否（占位保留） 自身原位置 ✓ 有效 微调位置；作为绝对定位子元素的参照父容器
absolute
绝对		 ✓ 是 最近已定位祖先 / body ✓ 有效 精确放置元素，配合"子绝父相"使用
fixed
固定		 ✓ 是 浏览器窗口 ✓ 有效 固定导航栏、悬浮按钮等始终可见的元素
static 静态定位
默认状态，按文档流排列，边偏移无效，无需主动声明。
relative 相对定位
相对自身原位置偏移，不脱流，原位置保留。常用作父容器参照。
absolute 绝对定位
脱离文档流，参照最近已定位祖先。配合子绝父相使用，精确放置。
fixed 固定定位
脱离文档流，始终参照浏览器窗口，滚动页面时位置不变。
