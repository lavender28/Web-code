§01 表单的构成

表单是网页与用户之间的"对话窗口"。通过表单，用户可以输入信息并提交给服务器，实现注册、登录、搜索、下单等功能。一个完整的表单由三个部分构成：

结构图 · 表单的三层组成
① 表单域 <form> — 容器，负责采集和传输数据
如果没有表单域，数据无法发送到服务器
② 提示信息（<label>）— 引导用户填写
说明性文字，告诉用户该填什么内容
③ 表单控件（<input> / <select> 等）— 用户操作的元素
文本框、单选按钮、复选框、提交按钮……
组成部分标签作用
表单域<form>整个表单的容器，负责采集并传输数据；没有表单域，数据无法发送到服务器
表单控件<input> <select> <textarea>具体的交互元素，如文本框、密码框、复选框、提交按钮等
提示信息<label> 或纯文字说明性文字，引导用户正确填写
§02 创建表单 <form> 三个核心属性
语法格式
<form action="url地址" method="提交方式" name="表单名称">   各种表单控件   </form>
html · form 基础结构
<form action="login.php" method="post" name="loginForm"> <!-- 各种表单控件写在这里 --> <input type="text" name="username"> <input type="submit" value="登录"> </form>
三个核心属性详解
属性 1action — 指定数据提交地址

action 属性指定接收并处理表单数据的服务器程序地址。属性值可以是相对路径、绝对路径，也可以是邮件地址。

html
<form action="form_action.asp"> <!-- 提交到服务器脚本 --> <form action="mailto:admin@example.com"> <!-- 提交到邮件地址 -->
属性 2method — 设置提交方式
GET 默认值
  • ✗数据附在 URL 后面，地址栏可见
  • ✗保密性差（密码会暴露在地址栏）
  • ✗有数据量限制（约 2KB）
  • →适合：搜索、简单查询
POST 推荐
  • ✓数据隐藏在请求体，地址栏不可见
  • ✓保密性好，适合传输密码等敏感数据
  • ✓无数据量限制
  • →适合：登录、注册、提交订单
属性 3name — 为表单命名

为表单指定唯一名称，便于 JavaScript 脚本识别和操作该表单。

html
<form name="registerForm">
§ 03 input 控件 最重要的表单控件

<input> 是表单中最常用的控件，是一个单标签。通过改变 type 属性的值，它能呈现出十余种完全不同的形态——文本框、密码框、单选按钮、复选框、提交按钮……

语法格式
<input type="控件类型" />
§ 03 type 属性详解 点击卡片展开说明
文本类
type="text"单行文本框
最常用的输入控件。常用于姓名、账号、证件号等简短信息。
常配合 value（默认值）和 maxlength（最大字数）使用。
type="password"密码框
输入内容以圆点显示，保护隐私。常用 size 控制显示宽度（字符数）。
type="hidden"隐藏域
对用户完全不可见，通常用于传递后台需要的参数（如 token、用户 ID），不需要用户填写。
（不显示任何内容）
选择类
type="radio"单选按钮
用于单项选择。关键：同组选项的 name 必须相同，浏览器才会实现"只能选其一"的约束。用 checked 设置默认选中。
type="checkbox"复选框
用于多项选择。可以同时选中多个选项，用 checked 设置默认选中。
⚠ 单选按钮的关键规则：同一组单选按钮的 name 属性值必须完全相同，才能实现互斥的"单选"效果。如果 name 不同，每个按钮相互独立，可以全部选中。
html · radio 与 checkbox 示例
<!-- 性别单选：name 相同，实现二选一 --> <input type="radio" name="sex" checked="checked"> 男 <input type="radio" name="sex"> 女 <!-- 兴趣复选：可多选 --> <input type="checkbox"> 唱歌 <input type="checkbox"> 跳舞 <input type="checkbox"> 游泳
按钮类 & 文件类
type 值功能说明
submit提交按钮表单核心控件，点击后将数据发送给服务器；可用 value 自定义按钮文字
reset重置按钮清空表单中所有已填内容，恢复默认值
button普通按钮不触发表单提交，通常配合 JavaScript 使用
image图像提交按钮功能与 submit 相同，但用图片替代按钮外观；必须指定 src 属性
file文件域显示文本框和"选择文件"按钮，供用户上传文件

§ 04 textarea 控件 — 多行文本框

<textarea> 用于创建多行文本输入框，适合输入大段文字，如个人简介、留言、评论等。它是一个双标签，标签之间的文字作为默认内容显示。

html · textarea 语法
<textarea cols="40" rows="5"> 这里是默认文字（可以为空） </textarea>
效果预览 · textarea 控件
个人简介：
属性描述
cols每行字符数（控制宽度）
rows显示行数（控制高度）
name控件名称
readonly只读
disabled禁用
💡 实际开发建议：各浏览器对 cols 和 rows 的理解不同，显示可能有差异，推荐用 CSS 的 width 和 height 来控制多行文本框的尺寸，更稳定一致。
§ 05 select 控件 — 下拉菜单

<select> 用于创建下拉菜单，适合选项较多、版面有限的场景（如省份、年份、职位）。每个选项用 <option> 标签定义。

效果预览 · select 下拉菜单

html · select 语法
<select> <option>北京</option> <option selected="selected">上海</option> <!-- 默认选中 --> <option>广州</option> <option>深圳</option> </select>
标签 / 属性描述
<select>下拉菜单容器
<option>每个选项，每个 <select> 至少需要一对

<select multiple="multiple">允许多选（按住 Ctrl 键可同时选择多项）
<option selected="selected">设置该项为默认选中
§ 06 HTML5 新的 input 控件类型 点击卡片展开详情

HTML5 为 <input> 扩展了多种新的 type 值。这些新类型能自动验证输入格式、提供原生交互组件（日期选择器、颜色选择器），大幅减少 JavaScript 验证代码。

type="email"
邮件地址
✅ 自动验证格式
自动检查输入内容是否符合 Email 格式（含 @ 符号）。不符合时提交会显示提示。

type="url"
URL 地址
✅ 自动验证格式
自动验证是否为合法的 URL 格式，必须以 http:// 或 https:// 开头。

type="tel"
电话号码
— 配合 pattern
本身不验证格式，通常配合 pattern 属性使用正则表达式验证。在手机端会呼起数字键盘。

type="number"
数字输入框
✅ 验证是否为数字
只允许输入数字，可配合 min、max、step 限制范围和步长。

type="range"
滑动条
— 范围数值选择
以滑动条形式显示，适合音量、亮度、评分等范围选择场景。

type="color"
颜色选择器
— 可视颜色选择
点击后弹出颜色选择器，返回 #RRGGBB 格式的颜色值。

type="date"
日期选择
— 日历组件
点击后显示原生日历选择器，返回 YYYY-MM-DD 格式。还有 month（年月）、week（周）、time（时分）等变体。

type="search"
搜索框
— 语义化搜索
语义上表示搜索输入，部分浏览器会显示清除按钮（×）。能自动记录搜索历史。

type="datetime-local"
本地日期时间
— 日期+时间
同时选择日期和时间（本地时区），比 date 更完整。

💡 浏览器兼容性提示：对于不支持新类型的旧版浏览器，控件会自动降级为普通文本框，不影响功能使用。

§ 08 新的 input 控件属性

HTML5 为 <input> 扩展了多个实用属性，进一步增强了表单的交互性和数据验证能力，大幅减少对 JavaScript 的依赖。

#属性功能
1autofocus页面加载完成后，该输入框自动获取焦点（光标自动定位）
2form将控件关联到页面任意位置的 <form>（通过 form 的 id）
3list绑定 <datalist> 提示列表（值为 datalist 的 id）
4multiple允许输入/选择多个值（适用于 email 和 file 类型）
5min / max / step设置数字或日期类型的最小值、最大值和步长
6pattern用正则表达式验证输入内容是否合法
7placeholder输入框内的灰色占位提示文字（输入后消失）
8required必填项，为空时不允许提交表单
重点属性详解
属性 7placeholder — 占位提示文字
html
<input type="text" placeholder="请输入用户名"> <input type="password" placeholder="请输入密码（6-18位）">
⚠placeholder 适用于：text、search、url、tel、email、password 类型。
属性 8required — 必填验证

属性 6pattern — 正则表达式验证

通过正则表达式验证用户输入是否符合特定格式，是前端数据验证的重要手段。

html · pattern 示例
<!-- 验证11位手机号 --> <input type="tel" pattern="^1[3-9]\d{9}$" placeholder="请输入11位手机号"> <!-- 验证6位数字密码 --> <input type="password" pattern="\d{6}" placeholder="请输入6位数字密码">
常用正则表达式速查
正则表达式说明
^[0-9]*$纯数字
^\d{6}$恰好 6 位数字
^\d{m,n}$m 到 n 位的数字
^[A-Za-z]+$纯英文字母
^[A-Za-z0-9]+$英文字母和数字的组合
^[\u4e00-\u9fa5]{0,}$纯汉字
^\w+([-+.]\w+)*@\w+([-.]w+)*\.\w+([-.]w+)*$Email 地址
^\d{15}|\d{18}$身份证号（15 位或 18 位）
^[a-zA-Z][a-zA-Z0-9_]{4,15}$账号（字母开头，5-16位，含字母/数字/下划线）
^[a-zA-Z]\w{5,17}$密码（字母开头，6~18位，含字母/数字/下划线）
§ 09 新的表单属性（<form> 标签上）

以下属性写在 <form> 标签上，控制整个表单的提交行为。

autocomplete自动完成
控制浏览器是否根据历史记录自动填充表单。
autocomplete="on"（默认）：允许自动填充
autocomplete="off"：禁止自动填充

适用场景：银行、医疗等安全性要求高的表单应设为 off。
novalidate跳过验证
提交表单时跳过所有 HTML5 内置格式验证（required、pattern 等）。

适用场景：开发调试阶段，或表单验证完全由 JavaScript / 后端控制时。
html
<!-- 开启自动填充（默认）--> <form autocomplete="on"> <!-- 禁止自动填充（涉密表单推荐）--> <form autocomplete="off"> <!-- 提交时跳过所有 HTML5 验证 --> <form action="submit.asp" method="get" novalidate="novalidate">
§ 10 使用 CSS 控制表单样式

浏览器对表单控件有各自的默认样式，不同浏览器之间存在差异。使用 CSS 可以统一并美化表单的外观，让表单既有功能又美观。

css · 完整表单美化代码
/* 使用属性选择器精准定位不同类型的 input */ input[type="text"], input[type="password"], input[type="email"], input[type="tel"], textarea, select { width: 280px; padding: 8px 12px; border: 1px solid #ccc; border-radius: 4px; font-size: 14px; outline: none; /* 去掉默认蓝色聚焦边框 */ transition: border-color 0.3s; } /* 获取焦点时改变边框颜色 */ input:focus, textarea:focus, select:focus { border-color: #c0392b; } /* 美化提交按钮 */ input[type="submit"] { background-color: #c0392b; color: white; padding: 10px 24px; border: none; border-radius: 4px; cursor: pointer; font-size: 14px; transition: background-color 0.2s; } input[type="submit"]:hover { background-color: #922b21; }
