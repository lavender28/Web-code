§ 01 创建表格 三个核心标签嵌套使用

当页面需要展示有规律的数据时（如成绩单、课程表、价格对比），表格是最直观的呈现方式。HTML 表格由三个核心标签嵌套而成，理解它们的层级关系是学习表格的第一步。

标签英文全称职责嵌套规则
<table>Table定义整个表格的容器最外层，包裹所有内容
<tr>Table Row定义表格中的一行必须嵌套在 <table> 内
<td>Table Data定义行内的单元格必须嵌套在 <tr> 内
html · 基本语法
<table> <tr> <td>单元格1</td> <td>单元格2</td> </tr> <tr> <td>单元格3</td> <td>单元格4</td> </tr> </table>

⚠
重要规则：<tr> 标签中只能嵌套 <td> 或 <th> 标签，不可以直接在 <tr> 中输入文字，否则页面结构会错乱。
💡 记忆技巧：把表格想象成 Excel——<table> 是整张工作表，<tr> 是每一行，<td> 是每个格子。

§ 04 <td> 标签的属性

<td> 是控制粒度最细的标签，可以精确控制每一个单元格的样式。其中 colspan 和 rowspan 是最重要的属性，用于合并单元格。

属性名含义常用属性值重要程度
width单元格宽度像素值了解，用 CSS 替代
height单元格高度像素值了解，用 CSS 替代
align内容水平对齐left / center / right了解，用 CSS 替代
valign内容垂直对齐top / middle / bottom了解，用 CSS 替代
bgcolor背景颜色颜色值了解，用 CSS 替代
background背景图像URL了解
colspan横跨列数（水平合并）正整数⭐ 重点掌握
rowspan竖跨行数（垂直合并）正整数⭐ 重点掌握
📌
联动规律（容易踩坑）：
对某个 <td> 设置 width → 该整列所有单元格都会跟着同宽
对某个 <td> 设置 height → 该整行所有单元格都会跟着同高
§ 04 合并单元格 colspan · rowspan — 点击演示查看过程

合并单元格是表格操作中最重要的技能，分为两种方向：水平合并（跨列）和垂直合并（跨行）。操作分为固定的两步。

合并步骤
第一步：在保留的单元格上添加 colspan（横向合并列数）或 rowspan（纵向合并行数）属性
第二步：删除或注释掉被"吞并"的多余单元格（<td>）
水平合并 colspan — 合并同一行的多列
交互演示 · colspan 合并过程
已合并：跨 3 列的大格子（colspan="3"）
格子4 格子5 格子6
已合并 — 第一行的3个格子合并为1个，显示为宽格子
html · colspan 水平合并示例
<table border="1"> <tr> <!-- colspan="3" 表示这个格子横跨3列 --> <td colspan="3">跨3列的大格子</td> <!-- 原来第2、第3个 td 已删除 --> </tr> <tr> <td>格子4</td> <td>格子5</td> <td>格子6</td> </tr> </table>
垂直合并 rowspan — 合并同一列的多行
交互演示 · rowspan 合并过程
已合并：跨3行的格子
（rowspan="3"）		 第1行第2列
第2行第2列
第3行第2列
已合并 — 第一列的3个格子合并为1个高格子
html · rowspan 垂直合并示例
<table border="1"> <tr> <!-- rowspan="3" 表示这个格子竖跨3行 --> <td rowspan="3">跨3行的格子</td> <td>第1行第2列</td> </tr> <tr> <!-- 第2、第3行的第1列 td 已删除 --> <td>第2行第2列</td> </tr> <tr> <td>第3行第2列</td> </tr> </table>
⚠
合并注意事项：
① 合并后必须删除多余的 <td>，否则表格列数会超出，布局错乱
② 合并方向要确认：横向（同行多列）用 colspan，纵向（同列多行）用 rowspan
③ colspan 和 rowspan 是重点，其余 <td> 属性均可用 CSS 替代

§ 06 表格的结构标签 thead · tbody · tfoot

为了让表格具有更清晰的语义，HTML 提供了三个结构化容器标签，将表格划分为头部、主体和页脚三个区域，便于浏览器和开发者理解表格的内容层次。

结构图 · thead / tbody / tfoot 的层次关系
<table>
<thead> — 表格头部（列标题行）
<tfoot> — 表格页脚（合计 / 备注）⬅ 写在 tbody 前
<tbody> — 表格主体（实际数据行，可有多个）
标签描述位置规则数量限制
<thead>表格头部，通常放列标题行位于 <table> 内最前只能有 1 个
<tfoot>表格页脚，通常放合计行或备注位于 <thead> 之后，<tbody> 之前只能有 1 个
<tbody>表格主体，放实际数据行位于 <tfoot> 之后可以有多个
html · 使用结构标签的完整示例
<table border="1"> <thead> <tr> <th>科目</th> <th>平时分</th> <th>考试分</th> </tr> </thead> <tfoot> <!-- 注意：tfoot 写在 tbody 之前 --> <tr> <td>合计</td><td>83</td><td>197</td> </tr> </tfoot> <tbody> <tr><td>语文</td><td>28</td><td>62</td></tr> <tr><td>数学</td><td>30</td><td>65</td></tr> <tr><td>英语</td><td>25</td><td>70</td></tr> </tbody> </table>

§ 07 使用 CSS 控制表格边框 border-collapse 是关键

使用 CSS 的 border 属性可以为表格和单元格分别设置更丰富的边框样式（颜色、宽度、线型）。其中 border-collapse 是控制表格边框呈现的核心属性。

交互演示 · 切换 border-collapse 效果
品牌 型号 价格
苹果 iPhone 16 ¥ 5,999
华为 Mate 70 ¥ 5,499
当前：separate — 相邻边框分离显示，呈现双线效果
css · border-collapse 用法
table { border: 1px solid #333; border-collapse: collapse; /* 合并相邻边框为单线 */ } th, td { border: 1px solid #ccc; padding: 8px 12px; }
⚠
当 border-collapse: collapse 生效时，HTML 中设置的 cellspacing 属性会失效，两者不能同时生效。
另外：<tr> 标签不支持 border CSS 样式属性，只能对 <table>、<th>、<td> 设置边框。
§ 08 使用 CSS 控制单元格边距

控制单元格与内容的距离，以及相邻单元格之间的距离，是表格美化中最常见的需求。

需求方法注意
单元格内容与边框的距离（内边距） 对 <th> / <td> 设置 CSS padding ✅ 完全支持
相邻单元格之间的距离（外边距） 只能用 <table> 的 HTML 属性 cellspacing ⚠️ margin 对 td 无效
css · 使用 padding 控制单元格内边距
th, td { padding: 10px 16px; /* 上下10px，左右16px */ } /* ❌ 以下写法无效 — td 不支持 margin */ td { margin: 10px; /* 无效！ */ }
⚠
三个"没有"要记住：
① <th> 和 <td> 没有 margin 外边距属性
② <tr> 标签既没有 padding 也没有 margin
③ 要设置相邻单元格间距，只能用 <table cellspacing="...">
§ 09 使用 CSS 控制单元格宽高

使用 CSS 的 width 和 height 属性可以控制单元格的宽高。单元格有一个与众不同的联动特性，需要特别注意。

联动规律（重要）
设置某个 <td> 的 width → 该整列所有单元格宽度统一变化
设置某个 <td> 的 height → 该整行所有单元格高度统一变化
css · 控制单元格宽高
/* 设置第一列宽度（整列生效）*/ td:first-child { width: 120px; } /* 设置表头行高（整行生效）*/ th { height: 48px; }
