01 视频、音频嵌入技术概述

在网络传输速度越来越快的今天，视频和音频技术已被越来越广泛地应用在网页设计中——相比静态的图片和文字，多媒体能为用户提供更直观、丰富的信息体验。

HTML5 之前 vs HTML5 之后
😓 HTML5 之前：第三方插件
  • ✗依赖 Flash、Silverlight 等第三方插件
  • ✗代码冗长复杂，维护困难
  • ✗移动端不支持 Flash，体验差
  • ✗安全漏洞多，Flash 已于 2020 年退役
✅ HTML5：原生标签嵌入
  • ✓使用 <video> 和 <audio> 原生标签
  • ✓代码简单精炼，一行即可嵌入
  • ✓无需任何插件，浏览器原生支持
  • ✓移动端完美支持

html · 基础嵌入视频
<!-- 最简写法：仅 src 和 controls --> <video src="video/movie.mp4" controls="controls"> 您的浏览器不支持 video 标签 </video>
<video> 常见属性详解
src
属性值：文件路径 / URL
设置视频文件的地址，可以是本地相对路径，也可以是网络 URL。
controls
属性值：controls
添加播放控件（播放/暂停按钮、进度条、音量等）。省略此属性则视频不会显示控制条。
autoplay
属性值：autoplay
页面加载完成后自动播放。⚠️ 2018年后 Chrome 禁止有声自动播放，需配合 muted 属性才能自动播放。
muted
属性值：muted
静音播放。配合 autoplay 使用，可以实现静音自动播放（如背景视频）。
loop
属性值：loop
视频播放结束后重新循环播放，适合背景视频或教程循环场景。
poster
属性值：图像 URL
视频缓冲不足或加载前，显示指定的封面图片。提升视觉体验，避免黑屏。
preload
属性值：auto / metadata / none
控制视频在页面加载时的预加载行为：
auto：页面加载时全部预加载（消耗流量）
metadata：只预加载视频元数据（时长、尺寸等）
none：不预加载
注意：使用 autoplay 时，此属性会被忽略。
html · 完整属性示例
<!-- 背景视频：自动静音循环播放 --> <video src="video/bg.mp4" autoplay muted loop poster="images/cover.jpg" ></video> <!-- 普通视频：带控件，只预加载元数据 --> <video src="video/lesson1.mp4" controls preload="metadata" poster="images/lesson1-cover.jpg" > 您的浏览器不支持 HTML5 video 标签 </video>
⚠
Chrome 自动播放限制：2018 年 1 月起，Chrome 浏览器取消了对有声自动播放的支持。若需要自动播放，必须同时添加 muted 属性（静音播放），否则 autoplay 将不生效。

06 视频、音频文件的兼容性问题与 <source> 一次提供多种格式

虽然 HTML5 支持多种音视频格式，但并非所有浏览器都支持所有格式。例如，Safari 不支持 .ogg，旧版 IE 不支持 .webm。因此，为了让视频/音频在所有浏览器中都能正常播放，需要提供多个格式的备用文件。

解决方案
使用 <source> 标签为 <video> 或 <audio> 提供多个备用文件。浏览器会按顺序尝试，播放第一个它能支持的格式。
浏览器播放决策流程
Chrome 浏览器
→
尝试 .mp4 ✓ 支持，直接播放
旧版 Safari
→
尝试 .ogg → 不支持 → 尝试 .mp4 ✓ 播放
旧版 IE 9
→
尝试 .ogg → 不支持 → .webm → 不支持 → .mp4 ✓ 播放
html · source 多格式兼容 — 音频
<audio controls="controls"> <source src="music/song.ogg" type="audio/ogg"> <source src="music/song.mp3" type="audio/mpeg"> <!-- 以上都不支持时显示此文字 --> 您的浏览器不支持 audio 标签 </audio>
html · source 多格式兼容 — 视频
<video controls="controls"> <source src="video/movie.ogg" type="video/ogg"> <source src="video/movie.mp4" type="video/mp4"> <source src="video/movie.webm" type="video/webm"> 您的浏览器不支持 HTML5 video 标签 </video>
<source> 属性说明示例
src媒体文件的路径或 URLsrc="music/song.mp3"
type媒体文件的 MIME 类型。视频以 video/ 开头，音频以 audio/ 开头type="audio/mpeg"
type="video/mp4"
⚠ 浏览器对音视频还有编码格式要求：视频编码最好为 H.264，音频编码最好为 AAC。若文件格式正确但仍无法播放，可用"格式工厂"等工具转换编码格式。
07 调用网络音频、视频文件

除了使用本地文件，还可以直接在 src 属性中填入网络 URL 来调用远程服务器上的音视频文件，无需下载到本地。

html · 本地文件 vs 网络 URL
<!-- 调用本地文件（相对路径）--> <audio src="music/1.mp3" controls="controls">浏览器不支持 audio 标签</audio> <!-- 调用网络 URL（将 src 替换为完整的网络地址）--> <audio src="http://www.example.com/music/song.mp3" controls="controls" >浏览器不支持 audio 标签</audio>
获取网络文件 URL 的步骤
①
获取文件的 URL：在目标网页上找到音频或视频，右键 → 复制媒体地址 / 获取外部链接地址。
②
插入文件 URL：将复制的完整 URL 粘贴到代码中 src 属性的值中，即可引用该资源。
⚠
两个重要注意事项：
① 稳定性：调用网络 URL 时，若对方服务器出现问题（如文件被删除、服务器下线），则所引用的 URL 也会失效，导致页面无法播放。
② 版权：在网页中嵌入音频或视频文件时，一定要注意版权问题，尽量选择经过授权许可的音视频资源。
08 使用 CSS 控制视频的宽度和高度

视频嵌入后，默认会以视频文件的原始分辨率显示，可能导致视频过大超出页面范围，或与页面布局不协调。通过 CSS 的 width 和 height 属性可以控制视频在页面上的展示尺寸。

为什么要控制宽高？
设置宽高后，浏览器在加载页面时会提前确定视频的尺寸，为视频预留合适的空间，避免页面布局因视频加载完成后发生抖动，保证整体布局稳定。
html + css · 控制视频尺寸
<!-- HTML：添加 class 供 CSS 选中 --> <video class="my-video" src="video/movie.mp4" controls></video> /* CSS：设置宽高 */ .my-video { width: 640px; height: 360px; } /* 响应式写法：宽度自适应，保持比例 */ .my-video { width: 100%; max-width: 800px; height: auto; }
交互演示 · 不同尺寸的视频效果对比
▶
width: 480px
当前：480px — 中等尺寸，适合内容页面的主视频区域
⚠
重要：通过 CSS 缩放视频，只是改变了视频在页面上的显示尺寸，视频的原始文件大小并没有改变。如果视频文件体积过大，应使用"格式工厂"等视频处理软件对视频进行压缩，再嵌入网页，以提升页面加载速度。
css · 常用视频样式技巧
/* 全宽自适应视频（响应式必备）*/ video { width: 100%; height: auto; display: block; } /* 视频容器保持 16:9 比例 */ .video-wrap { position: relative; width: 100%; padding-top: 56.25%; /* 16:9 = 9/16 = 56.25% */ } .video-wrap video { position: absolute; inset: 0; width: 100%; height: 100%; }

制作步骤
Step 1
分析效果图
观察整体布局结构：播放器由专辑封面、歌曲信息、播放控件、进度条、播放列表组成。主体用 <div> 构建，音频用 <audio> 嵌入。
Step 2
制作页面结构（HTML）
用 <div> 搭建播放器容器和各子模块，嵌入 <audio> 标签加载音频文件，用 <source> 提供多格式兼容。
Step 3
定义 CSS 样式
为播放器设置深色背景、圆角、阴影，用 display:flex 布局各控件，用 CSS 自定义播放进度条样式，覆盖浏览器原生控件外观。
