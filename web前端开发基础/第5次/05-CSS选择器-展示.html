CSS 选择器

从样式规则到四种基础选择器，掌握 CSS 精准定位元素的核心技术
前置：HTML 基础结构 标记 · 类 · ID · 通配符 三种引入方式 · 优先级规则
§ 01 CSS 简介
定义
CSS（Cascading Style Sheets，层叠样式表）是用于控制网页外观的样式语言。在现代 Web 开发中，HTML5 负责网页结构，CSS3 负责网页样式，两者分工明确、相互配合。

如果把制作网页比作建房子：HTML 是房屋的砖块和结构（定义了哪里是墙壁、哪里是门）；CSS 则是房屋的装修（决定了墙漆什么颜色、门有多宽）。CSS 的核心作用就是告诉浏览器，HTML 元素应该以什么样的外观展示在屏幕上。

💡 早期 HTML 中，样式与结构混写在一起（如用 <font> 标签设置颜色），导致代码冗余、维护困难。CSS 的出现实现了结构与表现的分离，极大提升了开发效率和可维护性。
CSS 的四大优势
  • ①
    内容与样式分离：HTML 专心负责内容逻辑，CSS 专心负责视觉表现，代码更易于阅读和维护。
  • ②
    极高的复用性：一套 CSS 规则可应用到成百上千个网页。修改一个文件，整个网站风格瞬间统一更新。
  • ③
    响应式设计：CSS 可检测用户设备屏幕大小，自动调整网页布局，在任何设备上保持美观。
  • ④
    丰富的视觉效果：动画、阴影、渐变、过渡等效果，极大丰富了用户的交互体验。
CSS 发展版本
版本说明
CSS1最早版本，支持基础字体、颜色、边距等属性
CSS2增加了定位、z-index、媒体类型等特性
CSS2.1CSS2 的修订版，修复了诸多错误，是长期主流版本
CSS3模块化设计，新增圆角、渐变、动画、弹性盒等大量现代特性
§ 02 CSS 样式规则

一条完整的 CSS 样式规则由两大部分组成：选择器（Selector）和声明块（Declaration Block）。声明块由花括号 {} 包裹，其中每条声明由属性和属性值用冒号分隔组成。

p {
font-size: 18px;
color: blue;
}
选择器
属性（Property）
属性值（Value）
分号分隔多条声明
要素内容说明
选择器p作用于页面中所有 <p> 标签
属性font-size字体大小
属性值18px18 像素
属性color文字颜色
属性值blue蓝色
⚠
书写规范：
① 属性和属性值之间用冒号 : 分隔（不能用等号）
② 多条声明之间用分号 ; 分隔
③ 最后一条声明的分号可省略，但建议保留（便于扩展）
④ CSS 对大小写不敏感，但建议统一用小写
§ 03 CSS 样式的三种引入方式 点击标签切换查看

CSS 样式有三种引入 HTML 文档的方式，分别适用于不同的使用场景。

交互演示 · 三种引入方式对比
定义
通过 HTML 标签的 style 属性，直接在标签内部书写 CSS 样式。
html · 行内式示例
<p style="font-size:18px; color:blue;"> 行内式样式的使用示例 </p>
优点缺点
书写简单，即写即用样式与结构混写，不易维护
优先级最高（1000），可覆盖其他样式无法复用，每个标签都要单独写
适合临时覆盖某个元素样式不利于统一修改
⚠行内式适合调试时临时覆盖样式。日常开发中不推荐大量使用。
定义
将 CSS 样式集中写在 HTML 文件的 <head> 标签内，并定义在 <style> 标签之间。
html · 内嵌式示例
<head> <style type="text/css"> h1 { font-style: italic; color: #00F; text-decoration: underline; } p { font-size: 36px; color: #F00; } </style> </head>
优点缺点
样式与结构分离，便于管理仅对当前 HTML 文档有效
修改方便，一处修改全页生效多个页面需要重复编写
适合单页面项目样式不能被其他页面复用
定义
将所有 CSS 样式单独存放在 .css 文件中，通过 <link> 标签链接到 HTML 文档。
html · index.html
<head> <link href="style.css" rel="stylesheet" type="text/css"> </head>
css · style.css
@charset "utf-8"; /* 含中文时必须声明编码，避免乱码 */ p { font-family: "黑体"; font-size: 24px; color: #390; }
优点缺点
多个页面共享同一 CSS 文件需要额外维护单独的 CSS 文件
修改一次，全站同步生效首次加载需要额外网络请求
浏览器可缓存 CSS，后续访问更快路径配置出错会导致样式失效
✓链接式是实际项目中最推荐的方式，有利于代码复用和维护。
三种方式对比总结
引入方式书写位置作用范围优先级推荐程度
行内式标签的 style 属性内仅当前标签最高（1000）不推荐
内嵌式<head> 内的 <style> 标签中当前 HTML 文件中（100）单页面可用
链接式<head> 内的 <link> 标签所有链接该文件的页面低（10）★ 推荐
§ 04 CSS 样式的优先级

当同一个元素被多种方式设置了样式，CSS 通过优先级（权重）机制决定最终显示哪套样式。基本规则：行内样式 > 内嵌样式 > 链接样式。

style=""
1000
ID 选择器
#id
100
类选择器
.class
10
标记选择器
p / div…
1
通配符
*
0
权重值选择器类型示例
1000行内样式style="color:red"
100ID 选择器#header
10类选择器、伪类选择器.nav、:hover
1标记选择器、伪元素选择器p、::before
0通配符、子选择器、继承样式*、>、+
🔑
复合选择器的权重计算：各基础选择器权重叠加。
例：div p.highlight = div(1) + p(1) + .highlight(10) = 权重 12
💡 优先级口诀：权重相同时，后写的样式覆盖先写的（就近原则）；权重不同时，权重高的优先显示，与书写顺序无关。
§ 05 标记选择器 元素选择器 · 权重 1
定义
直接通过 HTML 的标签名称来选中页面上的元素。当需要给某种类型的所有标签设置统一样式时，标记选择器是首选。
css · 语法格式
标签名 { 属性1: 属性值1; 属性2: 属性值2; }
  • —
    全局性：会影响页面上所有匹配的标签。设置 div { border: 1px solid black; }，则每一个 div 都会带上边框。
  • —
    优先级低：权重仅为 1，很容易被类选择器（10）或 ID 选择器（100）覆盖，适合设置网站的"基础底色"。
html + css · 示例
<style type="text/css"> h1 { font-size: 16px; font-style: italic; color: #F00; } </style> <body> <h1>第一个 h1 标题</h1> <h1>第二个 h1 标题（也会变红）</h1> </body>
常用场景
使用场景示例
重置默认样式body { margin: 0; padding: 0; }
统一段落样式p { line-height: 1.8; font-size: 14px; }
统一链接颜色a { color: #333; text-decoration: none; }
统一标题字号h1 { font-size: 28px; color: #222; }
§ 06 类选择器 以 . 开头 · 权重 10
定义
类选择器就像给 HTML 元素贴上了一个自定义"标签"。在 CSS 中以英文句点 . 开头，后跟自定义名字，能让你精准挑选出想要装饰的一组元素。
css + html · 定义与应用
/* CSS 中定义 */ .类名 { 属性1: 属性值1; } /* HTML 中应用 */ <标签名 class="类名">内容</标签名> /* 一个元素可同时应用多个类（空格分隔） */ <p class="bluefont font1">此段落同时拥有两个类</p>
  • ✓
    灵活性：可以把同一个类应用到不同类型的标签上（<h1>、<p>、<div> 均可）。
  • ✓
    可复用性：写一次 CSS 样式，可在页面任何地方多次使用，代码非常整洁。
  • ✓
    优先级：权重分值为 10，高于标记选择器（1分），可覆盖基础标签样式。
  • ✓
    多类叠加：一个元素可同时拥有多个 class，多个类名之间用空格分隔，自动继承所有类的样式。
html + css · 完整示例
h1 { font-size: 16px; font-style: italic; } p { font-family: "黑体"; } .font1 { text-decoration: underline; } .bluefont{ color: #00F; } <h1>普通标题</h1> <h1 class="bluefont">蓝色标题</h1> <p>普通段落</p> <p class="bluefont font1">蓝色 + 下划线段落</p>
⚠
类名命名规范：
① 第一个字符不能是数字（1btn 非法，应改为 btn1）
② 类名区分大小写（NavBar 和 navbar 是不同的类名）
③ 一般使用小写字母，多词之间用连字符 - 分隔（如 main-content）
④ 类名应具有语义性，能反映用途（如 .highlight、.nav-link）
§ 07 ID 选择器 以 # 开头 · 权重 100 · 页面唯一
定义
通过为 HTML 元素设置唯一的 id 属性来精确定位并设置该元素的样式。ID 选择器具有唯一性，一个页面中每个 id 值只能出现一次。在 CSS 中以井号 # 开头。
css + html · 定义与应用
/* CSS 中定义 */ #id名 { 属性1: 属性值1; } /* HTML 中应用 */ <标签名 id="id名">内容</标签名>
  • ★
    唯一性：同一个页面中，每个 id 值只能赋予一个元素，不可重复。
  • ★
    精确性：只作用于那一个指定 id 的元素，不会影响其他元素。
  • ★
    权重高：权重为 100，高于类选择器（10）和标记选择器（1）。
html + css · 完整示例
p { font-size: 14px; color: #333; } #intro { font-size: 18px; color: #c00; font-weight: bold; } #copyright { font-size: 12px; color: #999; text-align: center; } <p id="intro">引言段落：18px 红色加粗</p> <p>普通段落：14px 深灰色</p> <p id="copyright">版权信息：12px 灰色居中</p>
ID 选择器 vs 类选择器
对比项ID 选择器类选择器
CSS 语法#id名 { }.类名 { }
HTML 应用id="id名"class="类名"
唯一性同页面唯一，只能用一次可在多个元素上重复使用
一个元素能用多少个只能有一个 id可同时拥有多个 class
权重10010
适用场景页面唯一关键区域（页头、页脚、主内容区）需要复用的通用样式
💡 实践建议：ID 选择器常用于配合 JavaScript 操作 DOM（document.getElementById()），样式控制更多推荐使用类选择器，这样可以更灵活地复用样式。
§ 08 通配符选择器 使用 * · 权重 0 · 作用全部元素
定义
使用星号 * 作为选择器，可以匹配页面中的所有 HTML 元素，是作用范围最广的选择器。
css · 最常见用法：全局样式重置
/* 清除所有元素的默认内外边距，使布局更易控制 */ * { margin: 0; padding: 0; box-sizing: border-box; }
  • —
    作用范围：页面中所有元素，无一例外。
  • —
    权重：0（所有选择器中权重最低），极容易被其他选择器覆盖。
  • —
    常见用途：清除浏览器默认的内外边距，作为 CSS 重置的起点。
为什么要清除默认样式？

浏览器对不同 HTML 元素有各自的默认样式（User Agent Stylesheet），不同浏览器的默认值略有差异，导致同一页面在不同浏览器中显示不一致。使用通配符选择器统一重置，可以消除浏览器差异，让布局从"零"开始。

元素浏览器默认样式
<body>上下左右各有 8px 的 margin
<h1>~<h6>有上下 margin 和加粗效果
<p>有上下 margin
<ul>、<ol>有 padding-left 和列表符号
⚠ 通配符选择器会作用于页面每一个元素，在元素数量非常多的大型页面中，过度使用可能影响性能。实际项目中常用专门的 CSS 重置库（如 Normalize.css）代替纯通配符重置。

选择器选用决策思路
需要设置样式时，问自己：
├── 需要清除所有元素的默认样式（重置）？
└── 使用【通配符选择器】 * { margin: 0; padding: 0; }
├── 要改变页面中"所有"某类标签的样式？
└── 使用【标记选择器】 p { color: gray; }
├── 要改变"某些特定"元素，且需要复用到多个元素？
└── 使用【类选择器】 .highlight { color: red; }
└── 要精确定位页面中"唯一一个"元素？
└── 使用【ID 选择器】 #main-title { font-size: 32px; }
引入方式选用决策思路
选择 CSS 引入方式时，问自己：
├── 只是临时测试某个元素的样式？
└── 行内式（调试后建议移除）
├── 只有一个 HTML 文件，无需共享样式？
└── 内嵌式（写在 <head> 的 <style> 标签中）
└── 有多个 HTML 页面，需要统一风格？
└── 链接式（推荐，写在独立的 .css 文件中）
📌
学习建议：按照"理解概念 → 查看示例 → 动手实践 → 综合运用"的路径进行学习。CSS 的核心在于灵活组合，掌握优先级规则是解决样式冲突的关键。重点感受各类选择器作用范围的差异——"谁能命中这个元素、谁的权重更高"，是 CSS 学习的核心思维。
